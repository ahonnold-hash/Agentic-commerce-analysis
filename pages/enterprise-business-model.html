Layer 03 — Agentic Commerce Detail

Agent Discovery
Value Project JSON-LD schemas published at /value-model.json and /pricing-model.json. Linked from llms.txt at domain root. Discoverable by Perplexity, ChatGPT, Gemini, and purpose-built buying agents. Enables L1 discovery in the five-layer agent stack.
Value Authorization Engine
Before any agent transaction executes, the engine computes EVE against the agent's purchase intent. If the computed value/price ratio falls below the merchant-defined threshold, the transaction is declined with a structured reason code — equivalent to an issuer decline for insufficient value, not insufficient funds.
Post-Transaction Value Verification
After settlement, the engine tracks whether promised value drivers materialized. Outcome-linked pricing models use this data to trigger deferred payments, credits, or refunds. Feeds back into EVE model refinement — creating a continuously improving value model grounded in real transaction outcomes.
★ Value Project Integration: This entire layer is powered by Value Project schemas. Without machine-readable EVE and pricing models, agents have no structured data to query, the Value Authorization Engine has no threshold to enforce, and post-transaction verification has no baseline to measure against. The Value Project is not an add-on to this layer — it is the layer.

Layer 01 — Orchestration Infrastructure Detail

Why Proprietary vs. Licensed
Licensed orchestration (Spreedly, Primer, etc.) delivers the routing capability but leaves data ownership with the vendor. The intelligence feedback loop — transaction data → EVE inputs → better value models — requires data you own. Proprietary infrastructure is the prerequisite for Layers 2 and 3 to function. This is the foundational investment that makes everything else possible.
Direct-to-Scheme Economics
On $1B qualifying volume, D2S at +20 bps = $2M annual savings above Stage 2 orchestration baseline. Cloud-native Visa/MC APIs now implement in weeks. Acquirer spread entirely eliminated on eligible volume. Full end-to-end fee transparency. Enables real-time settlement and working capital gains. Per Visa Core Rules §4.1.24.2: agentic transactions cannot be processed in card-present environments — D2S applies to CNP eligible volume only.
Compliance Architecture
Per Visa Core Rules (April 2026): Agentic Payment Providers must enroll in VIC program (§4.1.24.1), obtain cardholder consent before storing credentials (§4.1.24.3), maintain order confirmations 120+ days (§4.1.24.8), and fall under the dedicated cardholder-responsibility framework introduced for agentic transactions (§4.1.24.10), whose liability allocation is still being worked out by legal analysts. These are incorporated into the orchestration compliance layer — not managed separately.

Layer 00 — Network Protocol Detail

What Networks Have Solved
All three networks have fully solved Layer 4 (transaction execution) in our five-layer agent stack. TAP verifies agent identity. Agentic Tokens scope credentials. ACE protects consumers. The merchant's orchestration layer hooks into all three via standardized APIs. The execution problem is done.
What Networks Have NOT Solved
Layers 1–3 and 5 of the agent stack — discovery, value evaluation, pricing negotiation, and outcome verification — are structurally absent from all network architectures. Confirmed by Visa's 923-page Core Rules: zero value evaluation capability exists anywhere in the rulebook. This isn't a payments-only gap — pricing researchers (Journal of Professional Pricing, Q1 2026) found no single framework solves value attribution in agent transactions. This is the white space Layers 2 and 3 above fill.
Cardholder Liability Warning
Visa Core Rules §4.1.24.10 establishes a dedicated cardholder-responsibility framework for agent-initiated transactions — legal analysts (Law360, May 2026) flag the precise liability allocation as still unsettled. What is settled: Zero Liability (§1.4.5.1) only covers unauthorized transactions, and a properly-consented agent purchase that delivers poor value is authorized, not fraudulent — meaning it falls outside Zero Liability's protection regardless of how §4.1.24.10 is ultimately interpreted. The Value Authorization Engine in Layer 3 is the primary consumer protection mechanism under Visa's current rulebook — because it prevents the bad-value transaction from ever executing, the liability question becomes largely moot.

Section 03

The Value Project — How It Integrates

The Value Project is not an add-on to this business model. It is the intelligence layer that makes every other layer commercially meaningful.

The Core Proposition

Payment networks have built the infrastructure for agents to pay. The Value Project builds the infrastructure for agents to decide. Without machine-readable EVE and pricing schemas, an agent arriving at your storefront can see that something costs $200. It cannot compute whether $200 is justified, how you compare to alternatives, or whether the transaction will deliver the cardholder's intended value. Under Visa §4.1.24.10, if the agent overpays, the cardholder falls under a dedicated responsibility framework whose liability allocation remains unsettled among legal analysts. The Value Project is the consumer protection mechanism that makes this exposure moot in the first place, by ensuring the overpayment never happens.

Value Model Schema (EVE)

Encodes NBCA reference value, PDV driver decomposition across six value categories, NDV cost offsets, net differentiation value, pricing range, and customer segment scoping. All as typed JSON-LD objects with formulas, variable declarations, confidence levels, and source citations. Computable by agents in real time.

Pricing Model Schema

Encodes pricing architecture type, value metric, tier structure, volume discount rules, contingent pricing conditions, and input schema for price computation. All as typed objects — not prose. Agent can compute an exact price quote for any configuration from the schema alone, without calling a human or external API.

Rubric Compliance Targets

Steven Forth's pricing rubric (80 pts) and EVE rubric (80 pts). Minimum passing score: 44 per rubric. Production-ready: 72+. A typical enterprise pricing page scores 16/160 combined — F grade. This architecture targets 130+ combined — B grade — in the first 90 days, A grade at 180 days.

Publication & Discovery

Schemas published at canonical URLs with validFrom, schemaVersion, and @context declarations. Listed in llms.txt at domain root. Fed into Visa ICC catalog discovery layer. Registered in PulseMCP (16,000+ agents). Discoverable by Perplexity, ChatGPT, Gemini, and commercial buying agents.

Next Best Competing Alternative (NBCA)

What Is the Customer's Realistic Alternative?

The NBCA is not a theoretical worst case — it is the specific, realistic alternative the customer would actually choose if they declined this program. For this segment, that is:

NBCA: Competitor Mid-Tier Chairs + Internal Management
Mid-tier ergonomic chairs from a competing supplier or online channel + internal facilities team manages delivery and installation + reactive maintenance (no quarterly service program).

The NBCA becomes the pricing floor anchor. Every dollar of value above the NBCA cost is available for shared value capture between merchant and customer.

Section 08

Risk Register

Honest risk assessment including the cost of inaction — the most frequently omitted risk in enterprise business cases.

Risk Level Description Mitigation
Cardholder Liability Exposure (Visa §4.1.24.10) High Cardholders fall under a dedicated responsibility framework for agent errors, with liability allocation flagged as unsettled by legal analysts (Law360, May 2026). Zero Liability does not cover properly-consented bad-value purchases regardless of how that allocation resolves. The organization may not be aware of this rules-verified exposure. Legal review immediately. Value Authorization Engine as primary mitigation — prevents bad-value transactions before execution. Amex ACE integration provides additional protection for Amex cardholders.
Value Project Schema Novelty Medium Standard announced May 2026 — immature and will evolve. Early schemas may require revision as community develops best practices. Industry-specific EVE driver vocabulary not yet standardized. Participate in the open-source community to influence direction. Design schemas modularly for easy updates. Pilot-first before portfolio investment. Treat early schemas as living documents, not one-time deliverables.
Organizational Resistance to Pricing Governance High Business unit pricing autonomy is culturally entrenched. Value Intelligence Owner role will challenge existing authority structures. Commercial teams may resist EVE-anchored discount discipline. Executive sponsorship before launch. Frame as enabling faster deals, not constraining sales. Show early wins where value-anchored pricing outperformed gut-feel pricing. Value Intelligence Owner must have genuine cross-functional authority — not just advisory role.
Agentic Commerce Timeline Risk Low AI agent purchasing timelines may extend beyond 2027–28 projections. If inflection is delayed, infrastructure built for agentic flows delivers reduced near-term ROI. Orchestration savings and value pricing discipline deliver standalone value independent of agentic commerce timing. Agentic layer is an option on future upside — not the primary investment thesis. Both streams justify investment regardless of agent timeline.
Value Attribution Complexity Medium When multiple factors (price change, AI agent, sales campaign, market conditions) contribute to an outcome, there is currently no rigorous method to allocate credit between them. Outcome-based pricing depends on solving this. Steven Forth / Michael Mansard / Edward Wong research actively working on this. Cooperative game theory (Shapley value) + causal models + temporal difference learning is the proposed solution framework. Hybrid pricing (base + outcome variable) with floors and caps is the practical mitigation while attribution science matures.
Cost of Inaction High Competitors building this capability now will have 12–24 months of data advantage by 2027–28. In agent commerce, first-mover schema publishers will have preferential agent discovery visibility that late entrants cannot easily overcome. AI-driven traffic to US retail already up 4,700% YoY. There is no mitigation for inaction — only a decision about how large the disadvantage becomes. The first-mover window on Value Project adoption is open now. Every quarter of delay is a quarter of compounding disadvantage in agent discovery algorithms.
